# 정렬

정렬 sort은 데이터를 정해진 기준에 따라 배치해 의미 있는 구조로 재설정하는 것을 말한다.


### 버블 정렬 (Bubble Sort)
- **설명**: 데이터의 인접 요소끼리 비교하고, swap 연산을 수행하며 정렬하는 방식. 배열이 정렬될 때까지 반복적으로 인접한 요소들을 비교하고 필요한 경우 위치를 바꿈.
- **시간 복잡도**: 최악의 경우와 평균의 경우 O(n²), 최선의 경우 O(n)
- **공간 복잡도**: O(1) (in-place)
- **연산 속도**: 일반적으로 느린 편, 이미 정렬된 데이터에 대해서는 빠를 수 있음.

### 선택 정렬 (Selection Sort)
- **설명**: 대상에서 가장 크거나 작은 데이터를 찾아가 선택을 반복하면서 정렬하는 방식. 각 반복에서 최소값(또는 최대값)을 찾아 해당 위치와 교환.
- **시간 복잡도**: 항상 O(n²)
- **공간 복잡도**: O(1) (in-place)
- **연산 속도**: 일반적으로 버블 정렬보다는 빠르지만, 대규모 데이터셋에는 비효율적.

### 삽입 정렬 (Insertion Sort)
- **설명**: 대상을 선택해 정렬된 영역에서 선택 데이터의 적절한 위치를 찾아 삽입하면서 정렬하는 방식. 소규모 데이터셋 또는 거의 정렬된 데이터에 효과적.
- **시간 복잡도**: 최악의 경우 O(n²), 최선의 경우 O(n)
- **공간 복잡도**: O(1) (in-place)
- **연산 속도**: 소규모 데이터셋에서는 빠르나, 대규모 데이터셋에서는 느림.

### 퀵 정렬 (Quick Sort)
- **설명**: Pivot 값을 선정해 해당 값을 기준으로 정렬하는 방식. Pivot을 기준으로 배열을 분할하고, 각 부분 배열을 재귀적으로 정렬.
- **시간 복잡도**: 평균과 최선의 경우 O(n log n), 최악의 경우 O(n²)
- **공간 복잡도**: O(log n) (재귀 호출 스택)
- **연산 속도**: 평균적으로 매우 빠르나, 최악의 경우 느릴 수 있음.

### 병합 정렬 (Merge Sort)
- **설명**: 이미 정렬된 부분 집합들을 효율적으로 병합해 전체를 정렬하는 방식입니다. 배열을 반으로 나누고, 각각을 정렬한 후 병합합니다.
- **시간 복잡도**: 모든 경우에 O(n log n)
- **공간 복잡도**: O(n)
- **연산 속도**: 대규모 데이터셋에서도 안정적으로 빠릅니다.

### 기수 정렬 (Radix Sort)
- **설명**: 데이터의 자릿수를 바탕으로 비교해 데이터를 정렬하는 방식입니다. 각 자릿수별로 데이터를 그룹화하고 정렬합니다.
- **시간 복잡도**: O(nk) (여기서 n은 데이터 수, k는 자릿수)
- **공간 복잡도**: O(n + k)
- **연산 속도**: 자릿수가 적은 데이터셋에서 매우 빠릅니다.
