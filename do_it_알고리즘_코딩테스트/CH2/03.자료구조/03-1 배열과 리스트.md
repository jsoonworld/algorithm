# 📊 03-1 배열과 리스트

기본 자료구조인 배열과 리스트는 비슷한 점도 많지만, 서로 다른 특징도 많다. 
두 자료구조의 특징을 정확하게 이해하고 문제가 요구하는 조건에 맞게 적절하게 선택해 사용하는 것이 중요하다.

## 🧠 배열과 리스트의 핵심 이론

### 📝 배열

배열은 메모리의 연속 공간에 값이 채워져 있는 자료구조다. 배열의 값은 인덱스를 통해 참조할 수 있으며, 선언한 자료형의 값만 저장할 수 있다.

#### 배열의 특징

- 인덱스를 사용하여 값에 바로 접근할 수 있다.
- 새로운 값을 삽입하거나 특정 인덱스에 있는 값을 삭제하기 어렵다. 값을 삽입하거나 삭제하려면 해당 인덱스 주변에 있는 값을 이동시키는 과정이 필요하다.
- 구조가 간단하여 코딩 테스트에서 많이 사용된다.

### 📜 리스트

리스트는 값과 포인터를 묶은 노드들을 포인터로 연결한 자료구조이다.

#### 리스트의 특징

- 인덱스가 없어서 값에 접근하려면 Head 포인터부터 순서대로 접근해야 하며, 이로 인해 접근 속도가 느리다.
- 포인터로 연결되어 있어 데이터 삽입이나 삭제 시 연산 속도가 빠르다.
- 선언 시 크기를 별도로 지정할 필요가 없으며, 크기가 동적으로 변한다.
- 포인터를 저장할 공간이 필요하여 배열보다 구조가 복잡하다.

# ⏲️ 리스트와 배열의 시간 복잡도

리스트와 배열은 자료 구조에서 중요한 역할을 하며, 각각의 시간 복잡도는 이들을 언제 사용해야 하는지를 결정하는 중요한 요소다.

## 📚 배열(Array)의 시간 복잡도

### 배열의 특징
- 배열은 데이터가 메모리 상에서 연속적으로 위치해 있어, 인덱스를 통한 접근이 매우 빠르다.

### 배열의 시간 복잡도
1. **접근(Access):** O(1)
    - 인덱스를 통해 각 요소에 바로 접근할 수 있다.
2. **삽입/삭제(Insertion/Deletion):** O(n)
    - 배열에서 특정 위치에 요소를 삽입하거나 삭제할 때, 나머지 요소들을 이동해야 하므로 시간이 더 걸린다.

## 📖 리스트(List)의 시간 복잡도

### 리스트의 특징
- 리스트는 데이터가 포인터를 통해 서로 연결되어 있어, 동적 크기 조정이 가능하다.

### 리스트의 시간 복잡도
1. **접근(Access):** O(n)
    - 리스트는 시작점에서 순차적으로 요소를 탐색해야 하므로 접근 시간이 더 걸린다.
2. **삽입/삭제(Insertion/Deletion):** O(1) (단, 삽입/삭제 위치에 이미 접근한 경우)
    - 이미 접근한 위치에서의 삽입이나 삭제는 매우 빠르게 진행된다. 하지만 특정 위치를 찾기 위해서는 O(n)의 시간이 소요될 수 있다.
