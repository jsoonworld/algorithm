# 📚 01-1 시간 복잡도 표기법 알아보기

알고리즘에서 시간 복잡도는 주어진 문제를 해결하기 위한 연산 횟수를 말한다.
일반적으로 `수행 시간은 1억 번의 연산을 1초의 시간으로 간주하여 예측한다.`

## 🕒 시간 복잡도 정의하기

### 시간 복잡도 유형

- 빅-오메가 (Big-Ω): 최선일 때(best-case)의 연산 횟수를 나타낸 표기법
- 빅-세타 (Big-Θ): 보통일 때(average case)의 연산 횟수를 나타낸 표기법
- 빅-오 (Big-O): 최악일 때(worst case)의 연산 횟수를 나타낸 표기법

### 빅-오 표기법의 시간 복잡도 비교
- O(log n): 로그 시간 복잡도, 매우 빠르게 증가
- O(n): 선형 시간 복잡도, 일정한 비율로 증가
- O(n log n): 선형 로그 시간 복잡도, n에 대해 로그만큼 더 증가
- O(n^2): 제곱 시간 복잡도, n이 증가함에 따라 제곱으로 증가
- O(2^n): 지수 시간 복잡도, 매우 빠르게 증가
- O(n!): 팩토리얼 시간 복잡도, 가장 빠르게 증가

# ⏱️ 01-2 시간 복잡도 활용하기

문제의 시간 제한이 2초라면 2억 번 이하의 연산 횟수로 문제를 해결해야 한다.

- 연산 횟수는 1초에 1억 번 연산하는 것을 기준으로 생각한다.
- 시간 복잡도는 항상 최악일 때, 즉 데이터의 크기가 가장 클 때를 기준으로 한다.

### 🧮 연산 횟수 계산 방법
- 연산 횟수 = 알고리즘 시간 복잡도 X 데이터의 크기
